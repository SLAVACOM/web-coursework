# 🌱 Система управления садовым инвентарём

**Версия:** 1.0  
**Тип:** Веб-приложение MVC  
**Технологический стек:** Node.js + Express + MySQL + EJS + jQuery

---

## 📋 Описание проекта

Полнофункциональная система управления инвентарём садовых инструментов и оборудования для предприятий, учреждений, муниципальных служб и хозяйств. Система позволяет эффективно отслеживать наличие имущества, управлять заявками на выдачу инструментов и контролировать их статус.

### Основные сценарии использования:
- **Муниципальные учреждения** — управление парком коммунального оборудования
- **Сельскохозяйственные предприятия** — учёт и распределение инвентаря между работниками
- **Образовательные учреждения** — контроль инструментов для практических занятий
- **Строительные объекты** — учёт и бронирование оборудования на площадке
- **Частные хозяйства** — централизованное управление инструментами в коллективе

## 🏗️ Архитектура и сущности

### 1. **Пользователь (User)**

**Поля:**
- `id` — уникальный идентификатор
- `username` — имя пользователя (логин)
- `password` — хешированный пароль (bcryptjs)
- `role` — роль в системе (enum: admin, storekeeper, user)
- `created_at` — дата создания аккаунта

**Роли и права:**

#### 👨‍💼 **Администратор (admin)**
- Полный доступ ко всем функциям системы
- Управление пользователями (создание, удаление, смена ролей)
- Создание и редактирование инструментов
- Просмотр и изменение всех заявок
- Управление категориями
- Аналитика и отчёты

#### 📦 **Кладовщик (storekeeper)**
- Просмотр всего инвентаря
- Создание и редактирование инструментов
- Обработка заявок пользователей (одобрение, выдача, возврат, отклонение)
- Управление категориями инструментов
- Отслеживание статуса заявок

#### 👤 **Обычный пользователь (user)**
- Просмотр доступного инвентаря с фильтрацией
- Создание заявок на выдачу инструментов
- Просмотр истории своих заявок и их статусов
- Отслеживание готовности заказанного инструмента

---

### 2. **Инструмент / Оборудование (Tool)**

**Поля:**
- `id` — уникальный идентификатор
- `name` — название инструмента (обязательное)
- `category_id` — категория (FK на categories)
- `quantity` — общее количество единиц в наличии
- `condition` — состояние (enum: отличное, хорошее, удовлетворительное, плохое)
- `purchase_date` — дата покупки/поступления
- `price` — стоимость (в рублях)
- `description` — подробное описание, характеристики
- `created_at` — дата добавления в систему

**Связанные сущности:**
- Несколько изображений (tool_images) для каждого инструмента
- Заявки на выдачу (bookings)

**Возможности:**
- ✅ Добавление нового инструмента с несколькими фотографиями
- ✅ Редактирование характеристик (название, категория, состояние, цену, описание)
- ✅ Загрузка нескольких фотографий (до 10 штук)
- ✅ Удаление инструмента и его фотографий
- ✅ Отслеживание реального количества доступных единиц (с учётом активных заявок)
- ✅ Оценка состояния с визуальными индикаторами (значки, цветные бэджи)

**Пример:** 
```
Лопата штыковая
Категория: Лопаты
Состояние: Хорошее ✓
Всего: 5 шт.
Доступно: 2 шт. (3 на выдаче)
Цена: 850 ₽
Описание: Стандартная штыковая лопата для копания земли...
```

---

### 3. **Категория (Category)**

**Поля:**
- `id` — уникальный идентификатор
- `name` — название категории

**Примеры категорий:**
- Лопаты
- Грабли
- Шланги и полив
- Ножницы и секаторы
- Тяпки и мотыги
- Прочее

**Возможности:**
- ✅ Создание новых категорий (администратор, кладовщик)
- ✅ Удаление категорий
- ✅ Фильтрация инструментов по категориям
- ✅ Автоматическое отображение названия категории на карточках и в фильтрах

---

### 4. **Изображение инструмента (ToolImage)**

**Поля:**
- `id` — уникальный идентификатор
- `tool_id` — ID инструмента (FK на tools)
- `filename` — имя файла изображения (хранится в `public/uploads/`)
- `created_at` — дата загрузки

**Возможности:**
- ✅ Загрузка нескольких фотографий при создании/редактировании инструмента
- ✅ Галерея с миниатюрами на странице деталей
- ✅ Быстрое переключение между фотографиями (без перезагрузки)
- ✅ Удаление отдельных фотографий (с кнопкой удаления)
- ✅ Автоматическая оптимизация имён файлов (защита от коллизий)

**Поддерживаемые форматы:** JPG, JPEG, PNG, WebP (макс. 5 МБ на файл)

---

### 5. **Заявка на выдачу инструмента (Booking)**

**Поля:**
- `id` — уникальный идентификатор
- `user_id` — пользователь, подавший заявку (FK на users)
- `tool_id` — запрашиваемый инструмент (FK на tools)
- `quantity` — количество запрашиваемых единиц
- `status` — статус заявки (enum)
- `note` — примечание от пользователя (для чего нужен инструмент)
- `created_at` — дата создания заявки
- `updated_at` — дата последнего обновления статуса

**Статусы заявки и их значение:**

| Статус | Значение | Кем устанавливается | Действие |
|--------|----------|-------------------|---------|
| **⏳ Ожидает** | pending | Пользователь при создании | Заявка в очереди на рассмотрение |
| **✅ Одобрено** | approved | Кладовщик | Заявка одобрена, ждёт выдачи |
| **📦 Выдано** | issued | Кладовщик | Инструмент выдан пользователю |
| **↩️ Возвращено** | returned | Кладовщик при возврате | Инструмент возвращён в инвентарь |
| **❌ Отклонено** | rejected | Кладовщик | Заявка отклонена (нет в наличии, отказ и т.д.) |

**Статус-машина (workflow):**
```
pending → approved → issued → returned
   ↘       ↗ rejected
```

**Возможности:**
- ✅ Автоматическая проверка доступности (система не позволит забронировать больше, чем в наличии)
- ✅ Расчёт реального доступного количества (total - sum(approved + issued))
- ✅ Примечания к заявке (причина, сроки использования и т.д.)
- ✅ История заявок с датами и временем
- ✅ Фильтрация по статусам

**Пример заявки:**
```
ID: 42
Пользователь: Иван Петров (user)
Инструмент: Лопата штыковая
Количество: 2 шт.
Статус: Выдано 📦 (выдано 2 часа назад)
Примечание: Для уборки территории на участке №3
Создана: 15.05.2026, 09:30
Обновлена: 15.05.2026, 10:45
```

## 🎯 Ключевые функции системы

### Для всех пользователей

#### 🔍 **Поиск и фильтрация инвентаря**
- Полнотекстовый поиск по названию и описанию (с debounce 300ms)
- Фильтр по категориям (dropdown)
- Фильтр по состоянию (отличное, хорошее, удовлетворительное, плохое)
- Сортировка (по названию, количеству, состоянию)
- Отображение количества результатов
- Адаптивная сетка (автоматическая подстройка под размер экрана)

#### 📊 **Отображение деталей инструмента**
- Галерея фотографий с миниатюрами
- Основная информация (название, категория, состояние)
- Количество в наличии и доступное количество
- Цена (если указана)
- Дата покупки
- Полное описание

### Для обычных пользователей (user)

#### 📋 **Создание заявок**
- Выбор количества (с валидацией по доступности)
- Добавление примечания (для чего нужен инструмент)
- Проверка доступности в реальном времени
- Невозможно забронировать больше, чем в наличии

#### 📜 **История заявок**
- Просмотр всех своих заявок
- Фильтрация по статусам
- Отслеживание статуса каждой заявки
- Временная шкала (дата создания и обновления)

### Для кладовщиков (storekeeper)

#### 📦 **Управление заявками**
- Список всех заявок в системе
- Фильтрация по статусам (ожидают, одобрены, выданы и т.д.)
- Одобрение/отклонение заявок
- Выдача инструмента (смена статуса на "выдано")
- Регистрация возврата инструмента

#### 🛠️ **Управление инвентарем**
- Добавление новых инструментов (с несколькими фото)
- Редактирование характеристик (название, количество, состояние, цену)
- Удаление инструментов
- Загрузка/удаление фотографий
- Управление категориями

#### 📊 **Отслеживание статистики**
- Видит все активные заявки
- Видит доступное количество каждого инструмента (с учётом заявок)
- Может отследить историю изменений статусов

### Для администраторов (admin)

#### 👥 **Управление пользователями**
- Создание новых аккаунтов (пользователи, кладовщики, администраторы)
- Смена роли пользователя
- Удаление пользователей
- Просмотр списка всех пользователей с датами создания

#### 🔑 **Полный контроль**
- Все функции кладовщика
- Управление всеми заявками
- Модерирование инвентаря
- Управление категориями
- Просмотр аналитики системы

## 🚀 Технические возможности

### Безопасность
- ✅ **Хеширование паролей** — bcryptjs с солью
- ✅ **Сессии** — express-session с сохранением в памяти
- ✅ **Защита SQL-инъекций** — параметризованные запросы (mysql2)
- ✅ **Роль-базированный доступ** — middleware проверяет права на каждый route
- ✅ **Валидация данных** — на уровне форм и контроллеров
- ✅ **Защита от CSRF** — через проверку сессии

### Производительность
- ✅ **AJAX для фильтрации** — без перезагрузки страницы
- ✅ **Оптимизированные SQL-запросы** — with субзапросы для расчёта доступности
- ✅ **Кеширование категорий** — загружаются один раз при просмотре
- ✅ **Адаптивный дизайн** — работает на всех устройствах
- ✅ **Debounce на поиске** — 300ms задержка для снижения нагрузки

### База данных
- ✅ **UTF-8 кодировка** — полная поддержка кириллицы
- ✅ **Транзакции** — при удалении инструмента удаляются и все его изображения
- ✅ **Foreign Keys** — целостность данных
- ✅ **Индексы** — на часто используемые поля (tool_id, user_id, status)

### Интеграция с системами
- ✅ **RESTful API** — JSON endpoints для будущих приложений
- ✅ **Docker контейнеризация** — MySQL в Docker Compose для быстрого развёртывания
- ✅ **Логирование** — ошибки через Express error handler
- ✅ **Переменные окружения** — `.env` для конфигурации

## 📊 Примеры использования

### Сценарий 1: Муниципальное учреждение

```
Администратор создаёт учёт парка уборочных машин и инструментов.

Шаг 1: Администратор добавляет категории
  → Снегоуборочное оборудование
  → Метлы и лопаты
  → Ёмкости и инвентарь

Шаг 2: Кладовщик добавляет инструменты с фото
  Пример: Снежный плуг
  - Количество: 3 шт.
  - Состояние: Хорошее
  - Цена: 45000 ₽
  - Фото: 5 изображений

Шаг 3: Рабочий подаёт заявку
  - Нужен снежный плуг
  - Количество: 1 шт.
  - Примечание: Для уборки ул. Ленина

Шаг 4: Кладовщик одобряет и выдаёт
  - Проверяет наличие (есть 3, свободно 2)

```

```
Учитель биологии хочет провести практическое занятие.

Шаг 1: Просматривает доступный инвентарь
  - Фильтр по категории: "Инструменты для работы с растениями"
  - Видит лопаты, грабли, ведра, шланги
  - Отображение состояния и доступного количества

Шаг 2: Подаёт заявку на нужные инструменты

  - Проверяет наличие
  - Одобряет и помечает как "Выданы"

Шаг 4: Учитель забирает инструменты
  - Видит статус "Готово к выдаче"

Шаг 5: После занятия — возврат
  - Заведующий меняет статус на "Возвращено"
  - Инструменты снова доступны для других
```

## 🎨 Интерфейс и UX

### Дизайн-система
- **Цветовая схема:** Зелёная тема (экологично для "садового" контекста)
- **Основной цвет:** `#16a34a` (зелень)
- **Нейтральные:** градиент серых от `#f9fafb` до `#111827`
- **Статусы:**
  - ✅ Успех — зелёный
  - ❌ Ошибка — красный
  - ⏳ Ожидание — жёлтый
  - ❌ Отклонено — красный

### Адаптивность
- **Desktop (1100px+):** 4-5 колонок в сетке инструментов
- **Tablet (768-1100px):** 3-4 колонки
- **Mobile (< 768px):** 1-2 колонки, свёрнутое меню

### Навигация
- **Главная (/)** — Список инструментов с фильтрами (для всех)
- **/tools/:id** — Деталь инструмента с галереей и заявкой
- **/tools/new** — Создание инструмента (только для кладовщика+)
- **/tools/:id/edit** — Редактирование (только для кладовщика+)
- **/bookings** — Все заявки (только для кладовщика+)
- **/my-bookings** — Мои заявки (для обычных пользователей)
- **/categories** — Управление категориями (только для кладовщика+)
- **/admin/users** — Управление пользователями (только для администратора)
- **/login** — Вход в систему

## 🚀 Быстрый старт

### Локальная разработка

```bash
# 1. Клонировать репозиторий
git clone <repo>
cd web

# 2. Установить зависимости
npm install

# 3. Создать .env файл
cp .env.example .env
# Заполнить переменные окружения

# 4. Поднять БД в Docker
docker compose up -d

# 5. Запустить сервер в режиме разработки
npm run dev

# 6. Открыть в браузере
# http://localhost:3000

# 7. Войти с тестовыми учётными данными
# Администратор: admin / admin123
# Кладовщик: sklad / sklad123
# Пользователь: guest / guest123
```

