## 1. ВВЕДЕНИЕ

Развитие информационных технологий и широкое распространение интернета в современном мире существенно изменили подходы к ведению хозяйства и управлению имуществом. Одной из сфер, которая активно трансформируется под влиянием этих изменений, является управление инвентарём и распределение оборудования в организациях. В частности, новые подходы к организации учёта и предоставлению доступа к информации об имуществе открывают возможности для эффективного контроля ресурсов и оптимизации операционной деятельности. Данная курсовая работа посвящена разработке клиент-серверного приложения для управления садовым инвентарём, способного обеспечить пользователям и администраторам удобный доступ к информации об оборудовании и инструментах, а также возможность организации системы заявок для рационального распределения ресурсов.

### 1.1 Обоснование выбора темы

Управление инвентарём и оборудованием является критически важной задачей для организаций различного масштаба и направления деятельности. Муниципальные учреждения, сельскохозяйственные предприятия, образовательные организации и строительные компании ежедневно сталкиваются с необходимостью отслеживания наличия инструментов, их состояния и распределения между работниками.

Традиционные методы ведения учёта (бумажные журналы, таблицы в MS Excel) имеют существенные недостатки:
- **Неэффективность** — ручной ввод данных занимает много времени и подвержен ошибкам
- **Уязвимость данных** — высокий риск потери информации при отсутствии резервного копирования
- **Отсутствие контроля** — сложно отследить реальное наличие и текущее распределение оборудования
- **Отсутствие аналитики** — невозможно анализировать тенденции использования и планировать закупки

Разработанное веб-приложение позволяет автоматизировать процесс управления инвентарём, обеспечить надёжное и безопасное хранение данных, организовать централизованный доступ для авторизованных пользователей и предоставить инструменты для аналитики и контроля.

### 1.2 Актуальность решения

В условиях цифровизации и повышающихся требований к эффективности операционной деятельности автоматизированные системы управления инвентарём становятся не просто желательны, а объективно необходимы для любой организации, работающей с материальными ресурсами. Система позволяет достичь следующих преимуществ:

- **Оптимизация времени** — сокращение времени на поиск нужного инструмента и обработку заявок
- **Точность учёта** — исключение ошибок при ручном ведении журналов и реестров
- **Контроль доступности** — предотвращение конфликтов при распределении ресурсов благодаря расчёту реального наличия
- **Прозрачность процессов** — обеспечение видимости всех операций с инвентарём для авторизованных пользователей
- **Возможность аналитики** — получение данных для анализа использования оборудования и планирования закупок
- **Масштабируемость** — возможность работы системы с растущим количеством инструментов и пользователей

- **База данных** — использование реляционной СУБД MySQL на стороне web-сервера для надёжного хранения данных;
- **Развёртывание** — возможность запуска ПО web-сервера на локальной машине или в контейнеризированной среде Docker;

## 3. АРХИТЕКТУРА СИСТЕМЫ

### 3.1 Описание компонентов

Приложение построено по паттерну **Model-View-Controller (MVC)** и включает следующие компоненты:

**Скрипты JS клиента** в зависимости от действий пользователя генерируют HTTP-запросы к серверу. На стороне браузера реализована логика работы с галереей изображений, динамического поиска и фильтрации инструментов через AJAX, а также визуальное обновление страницы без полной перезагрузки.

**Маршрутизатор** обрабатывает входящий HTTP-запрос и сопоставляет его с соответствующим действием контроллера. В приложении используются отдельные модули маршрутизации для каждой функциональной области: инструменты, заявки, категории, аутентификация и административный раздел.

**Контроллер** преобразует запрос к серверу в конкретное действие. Как правило, действие соотносится с операцией над базой данных через вызов соответствующего метода модели. После обработки данных контроллер формирует ответ: либо передаёт данные в шаблонизатор для рендеринга HTML-страницы, либо возвращает JSON для обработки на стороне клиента при AJAX-запросах.

**Модель** представляет собой объектную абстракцию элементов базы данных. Вся работа с MySQL сосредоточена исключительно в моделях: параметризованные SQL-запросы, получение, запись и обновление данных. Контроллеры напрямую взаимодействуют с моделями, не содержа SQL-логики.

**Представление** — в данном приложении это EJS-шаблоны на стороне сервера, которые компилируются в HTML и отправляются браузеру. При AJAX-запросах (поиск, фильтрация) контроллер отправляет данные в виде JSON непосредственно клиентскому скрипту, который самостоятельно обновляет DOM без перезагрузки страницы.

Декомпозиция взаимодействия маршрутизатора и контроллера представлена на рис. 3.1, структурная схема доступа по ролям — на рис. 3.2.

### 3.2 Схема потока данных

```
Браузер (jQuery/JS)
       |
       | HTTP GET / POST
       v
  Express Router
       |
       v
  Auth Middleware ──── нет прав ───► 403 Forbidden
       |
   есть права
       |
       v
   Контроллер
    /        \
   /          \
  v            v
Модель      res.render() / res.json()
  |               |
  v               v
MySQL DB      EJS View / JSON
               |
               v
           Браузер
```

### 5.2 Статус-машина заявок

```
pending ──► approved ──► issued ──► returned
   └──────► rejected
```

| Статус | Кто устанавливает | Смысл |
|--------|-------------------|-------|
| pending | Пользователь | Заявка создана, ожидает рассмотрения |
| approved | Кладовщик | Одобрена, ожидает выдачи |
| issued | Кладовщик | Инструмент выдан |
| returned | Кладовщик | Инструмент возвращён |
| rejected | Кладовщик | Заявка отклонена |

## 6. БЕЗОПАСНОСТЬ

- **Хеширование паролей** — bcryptjs, соль 10 раундов
- **Защита от SQL-инъекций** — только параметризованные запросы (`pool.execute()`)
- **Сессии** — express-session, проверка на каждом маршруте
- **RBAC** — middleware проверяет роль пользователя перед доступом к ресурсу
- **Валидация загрузок** — проверка MIME-типа и размера (макс. 5 МБ)

## 9. ЗАКЛЮЧЕНИЕ

Разработанная система управления садовым инвентарём представляет собой полнофункциональное клиент-серверное веб-приложение, реализованное на стеке Node.js + Express + MySQL + EJS. В ходе работы был получен практический опыт проектирования реляционных баз данных, разработки серверного REST API, реализации системы аутентификации и авторизации с ролевым управлением доступом, а также организации интерактивного пользовательского интерфейса с применением AJAX.

Приложение реализует полный цикл управления инвентарём: от добавления инструментов кладовщиком до подачи заявок пользователями и обработки их администратором. Система обеспечивает контроль доступности ресурсов в реальном времени и поддерживает прозрачный workflow заявок через пять статусов.
